# Contributing

Thanks for looking. ACP Devtools is an npm-workspaces monorepo: TypeScript
(strict, NodeNext), Node ≥ 22, Vitest, ESLint + Prettier.

## Build from source

```bash
git clone https://github.com/maksugr/acp-devtools.git
cd acp-devtools
npm install                     # also installs the pre-commit hook
npm run build:full              # builds core, cli, ui, embeds UI in cli dist
cd packages/cli && npm link     # exposes `acp-devtools` globally via symlink
which acp-devtools              # varies by your Node install location
```

Requires Node 22+ and macOS / Linux / WSL.

## Everyday commands

```bash
npm test               # vitest run — full suite across core / cli / ui
npm run lint           # eslint
npm run typecheck      # tsc --noEmit on core + cli + ui
npm run format         # prettier --write
npm run precommit      # lint + typecheck + test (also the git hook)
```

## Try it on fixtures

The repo ships seven demo sessions, each stressing a
different surface — a 1392-message marathon, a spec-violating session, a
streaming-heavy one, and so on. Seed them into your capture database and they
show up in the inspector's session picker:

```bash
npm run fixtures:seed          # generate + import all seven into captures.db
acp-devtools ui                # open the inspector and pick one
```

Generate just one instead:

```bash
node fixtures/generate-fat-session.mjs --out /tmp/fat.json
acp-devtools import /tmp/fat.json
```

## UI hot-reload during development

`npm run dev:ui` runs Vite with HMR on port 5173 and serves your capture
database through the same session picker, so component edits hot-reload while
you inspect a real (or seeded) session.

```bash
npm run dev:ui                 # → http://127.0.0.1:5173/  — pick a session
```

## Adding a new agent shortcut

1. Open `packages/core/src/agents/registry.ts`.
2. Add an entry:

   ```ts
   'your-shortname': {
       shortName: 'your-shortname',
       displayName: 'Your Agent',
       description: 'One-line description that shows up in `--agent` help',
       command: 'npx',                       // or a binary name on PATH
       args: ['-y', '@your-scope/your-acp'], // or ['acp']
       // requiresExternalInstall: true,      // if it's a binary, not an npm pkg
       // aliases: [['npx', '-y', 'old-name']], // for deprecated names
   },
   ```
3. Run `npm test`. The inspector turns an agent command into your `displayName`
   in the session picker; cover that in
   `packages/ui/src/lib/captureLabel.test.ts`:

   ```ts
   expect(shortAgentName('npx -y @your-scope/your-acp')).toBe('Your Agent');
   ```
4. Update the *Supported agents* table in the root `README.md`.
5. Open a PR.

```

                                 discovery, shared HTTP server, agents registry
  cli/     acp-devtools         commander CLI: proxy / replay / ui / doctor / …
  ui/      @acp-devtools/ui     Vite + React + Tailwind frontend

  zed-config.md                    Zed agent_servers setup
  jetbrains-config.md              WebStorm / IntelliJ / PyCharm setup
  claude-code-setup.md             Claude Code multi-profile recipe

docs/                              product docs
  cli.md  ui.md  mcp.md  recipes.md

fixtures/                          dev/test data (contributor-only)
  mock-agent.js                    scripted ACP agent for tests
  sample-session.jsonl             3-request input fixture
  ws-client.js                     minimal WS subscriber for terminal debugging
  fixture.db                       generated by `npm run fixture:generate` (gitignored)
  seed.mjs                         generate + import all fixtures (npm run fixtures:seed)
  generate-fat-session.mjs         1392 msg / 4 days / mixed idle gaps — UI stress test
  generate-broken-session.mjs      28 msg with ~14 ACP spec violations — validate UX
  generate-error-storm.mjs         9 errors across 6 methods — ERRORS insight + red rects
  generate-tiny-session.mjs        5 msg / 1 prompt — edge cases (N=1 percentiles)
  generate-streaming-heavy.mjs     1741 msg (1733 streaming chunks) — StreamCluster + BUSIEST
  generate-permission-flow.mjs     19 msg with 6 agent→editor requests — `agent-req` lane
  generate-protocol-edges.mjs      14 msg with parse errors + orphan frames — decoder edges
```

The `generate-*.mjs` scripts each emit a `SessionExport` JSON to stdout (or to
`--out PATH`); pipe through `acp-devtools import` to seed your database (or run
`npm run fixtures:seed` to do all of them at once):

```bash
node fixtures/generate-fat-session.mjs --out /tmp/fat.json
acp-devtools import /tmp/fat.json
acp-devtools ui                        # pick the imported session
```

## Packaging

The CLI is bundled with **tsup**. `@acp-devtools/core` and `@acp-devtools/ui`
are private workspace packages that never reach npm — `core` is inlined into the
CLI bundle, `ui` is embedded as a static bundle at `dist/ui/`. The published
`acp-devtools` tarball is exactly `dist/index.js` plus `dist/ui/`. Runtime deps
pulled from npm are `better-sqlite3`, `ws`, `commander`,
`@agentclientprotocol/sdk`, and `@modelcontextprotocol/sdk`.

## Pre-commit hook

`simple-git-hooks` is wired in via the root `prepare` script, so a fresh `npm
install` registers it. The hook runs `npm run precommit`. To skip it on a single
commit (genuine emergencies only): `SKIP_SIMPLE_GIT_HOOKS=1 git commit ...`.

## Conventions

- 4-space indentation everywhere (TS, JSON, configs, Markdown), enforced by
  Prettier + EditorConfig.
- ESM with explicit `.js` import extensions (NodeNext); `import type` for
  type-only imports.
- Tests are colocated `*.test.ts`; ship a test in the same change as the code.
- Comments only for non-obvious *why* — no narration of what the code does.
